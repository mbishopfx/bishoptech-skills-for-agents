# 🛡️ Runtime Guardrail Loop: Continuous Watchdog & Closed-Loop Reinforcement

> **"If your AI system has no tests, your users are the test suite."**

In traditional software, we write unit tests to ensure changes do not break existing code. In AI systems, prompts and models are non-deterministic: a minor prompt adjustment that improves case A can silently degrade cases B, C, and D.

`runtime-guardrail-loop` is an installable agent skill that brings software regression testing and continuous monitoring to LLM pipelines. It compares prompt/model modifications against a saved test suite and closes the loop by turning live production failures into permanent test cases.

---

## ⚡ Key Capabilities

- **Regression Testing (Pre-Ship)**: Measures candidates against baseline ground-truth test suites, preventing accidental performance regressions.
- **Production Watchdog (Post-Ship)**: Samples live execution logs on a timer to catch drift or hallucinations.
- **Closed-Loop Feedback**: Feeds flagged failures back into the testing suite automatically, raising the quality floor with every mistake.
- **Delta Analysis**: Provides explicit, quantitative comparisons (e.g. "Prompt change increased accuracy on edge case inputs by +0.12 but regressed markdown parsing by -0.08").

## 🚀 Quick Start & Installation

```bash
# Register the skill with your agent CLI
agents skill install file:///Users/matthewbishop/BishopTech.dev/bishoptech-skills-for-agents/runtime-guardrail-loop
```

### 1. Initialize the Test Suite
Edit `templates/test_suite.json` to define your core inputs and baseline targets.

### 2. Run a Pre-Ship Regression Check
To evaluate a prompt modification, execute the regression script:
```bash
python3 runtime-guardrail-loop/scripts/regression_check.py \
  --suite-path "runtime-guardrail-loop/templates/test_suite.json" \
  --candidate-prompt "Write a short summary of this ticket" \
  --api-key "$GEMINI_API_KEY"
```

### 3. Ingest a Production Failure (Closed-Loop)
When a user flags a bad output, close the loop immediately:
```bash
python3 runtime-guardrail-loop/scripts/closed_loop.py \
  --suite-path "runtime-guardrail-loop/templates/test_suite.json" \
  --input "Cancel my subscription immediately." \
  --bad-output "Your subscription has been renewed." \
  --reason "Hallucinated action; did not cancel."
```

## ⚙️ Closed-Loop Reinforcement Architecture

```
[User Session] ──► [Failure Flagged] ──► [closed_loop.py]
                                              │
                                              ▼
[Safe Deployment] ◄── [regression_check.py] ◄── [test_suite.json]
```

1. **The Bug**: A user flags a bad output (e.g., via a thumbs-down in your UI).
2. **The Capture**: `closed_loop.py` captures the original prompt input, the bad output generated by the LLM, and the reason for the failure, then serializes it into `test_suite.json`.
3. **The Test**: During the next development cycle, `regression_check.py` runs the modified prompt against this new case.
4. **The Gate**: The code cannot be deployed until the LLM's new output passes the test, ensuring the bug is resolved once and never reintroduced.
